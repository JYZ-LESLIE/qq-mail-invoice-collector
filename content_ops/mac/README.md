# 发票管家 Mac App

这是发票整理项目的原生 macOS 运行外壳。它不替代现有 Python 发票引擎，而是把账号选择、日期范围、运行状态、Excel 打开入口和人工复核入口放进一个 Mac 工具窗口。

## 使用方式

首次使用或修复环境时，双击项目根目录的：

`准备发票管家.command`

日常使用时，双击项目根目录的：

`启动发票管家.command`

准备脚本会自动：

1. 检查 Python 虚拟环境。
2. 安装发票整理依赖。
3. 确保私密配置模板存在。
4. 编译 `发票管家.app`。

启动脚本只负责打开已经准备好的 `发票管家.app`。如果 App 还不存在，它会自动调用准备脚本。

不要把 `发票管家.app` 单独拖走；当前版本需要留在项目根目录，通过启动脚本打开。

## 覆盖安装和数据安全

准备脚本和构建脚本只覆盖项目根目录下的 `发票管家.app` 程序本体，不会删除或重建 `发票整理/`。

以下数据目录都在 App 外面，覆盖安装时会保留：

- `发票整理/私密配置/`
- `发票整理/台账/`
- `发票整理/报销管理/`
- `发票整理/人工复核/`
- `发票整理/已整理发票/`
- `发票整理/原始附件/`

App 左下角和菜单栏“关于发票管家”会显示版本号。当前可识别版本是 `0.1.9`，build `20260520.5`。

## 验收模式

App 默认打开“演示模式，不连接邮箱”。演示模式只展示运行完成后的界面，不读取邮箱、不下载附件。关闭演示模式后，点击开始才会真正整理发票。

## UI 原则

- 首页就是运行中心，不做网页式欢迎页。
- 左侧是账号和功能入口，中间是任务控制、运行日志、最近台账。
- 顶部工具栏只放高频动作：开始、打开台账、人工复核。
- 使用 SwiftUI 原生控件和 macOS 26 的系统材质，让系统自动继承 Liquid Glass 风格。
- 发票台账和日志区域保持高可读性，不把数据表做成营销卡片。

## 架构

```text
发票管家.app
  -> SwiftUI 原生窗口
  -> 调用 content_ops/scripts/invoice_multi_account_runner.py
  -> 复用现有 IMAP、解析、去重、Excel 生成逻辑
```

生成的 `.app` 是本机产物，不提交到 GitHub；源码和构建脚本在 `content_ops/mac/`。
